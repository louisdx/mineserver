Installation Example: Linux

The configuration of mineserver might require some explanation, so let us walk through a typical build process on a Linux system.

First, we download the source by cloning the repository. Suppose we want to store the source code in /data/projects/, and we will build the binaries in the separate directory /tmp/msbuild (that way we never pollute the source directory).

# get the source code cd /data/projects git clone git://github.com/louisdx/mineserver # set up the build directory cd /tmp mkdir msbuild cd msbuild # run cmake to create the Makefile cmake /data/projects/mineserver # build make

The binary should now reside in ./bin/mineserver. The cmake process also copied the file config.cfg into the ./bin/ directory. If you like, you can edit that file now and keep all your configuration data and the world file in that location. Alternatively, and especially if you want to run several different builds of mineserver with the same configuration, you should keep all your configuration and map files in the same, central location, $HOME/.mineserver. This is the default if no other config file is found, so delete ./bin/config.cfg and start up the program to create the default config file in $HOME/.mineserver.

However, at this stage the server does not handle the setup process completely, so whichever way you decided, you will need to make sure that the recipes directory and the binary plugin files are located in the directory pointed to by the config file. For example, by default the config file says:

system.path.data = "files"; system.path.plugins = "plugins";

These paths may be either absolute (like /usr/share/mineserver/files), or relative (as in the default setting). Relative paths are relative to the location of the config file. So if your config file resides in $HOME/.mineserver/, then the above configuration tells mineserver to expect the additional configuration files (like the whitelist) in $HOME/.mineserver/files/, recipes in $HOME/.mineserver/files/recipes/, and plugins in $HOME/.mineserver/plugins/.

Please make sure that all those files are in the appropriate place! Default versions of the config files and recipes are located in your build directory in the ./bin/files/ subdirectory; the binary plugin files are in the ./plugins/bin/ subdirectory.
